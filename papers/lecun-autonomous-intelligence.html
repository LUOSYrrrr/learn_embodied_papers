§ 01

一句话总结

核心主张

LeCun 认为走向自主机器智能的路径，是构建一个模块化、完全可微分（fully differentiable）的系统， 其核心是世界模型（World Model）——让 AI 能在脑中"想象"行动后果， 而不是像当前 LLM 那样单纯做序列预测。

这篇技术报告并非在某个具体任务上取得 SOTA，而是提出了一套系统架构蓝图， 试图回答：机器要达到类人的常识推理和规划能力，缺少的是什么？

§ 03

架构概览

整个系统由六个模块组成，所有模块都被假设为可微分（differentiable）， 这意味着代价（Cost）的梯度可以反向传播（backpropagate）穿越所有模块，驱动学习。

图：LeCun 自主智能架构信息流（蓝色虚线 = Configurator 配置信号；红色 = 梯度反向传播）

所有模块之间通过箭头连接的方向，都可以反向传播梯度——这是这个架构的关键设计原则， 使得端到端可微分学习（end-to-end differentiable learning）成为可能。

§ 04

六大模块精读

Actor

配置器

CONFIGURATOR MODULE

接收所有其他模块的输入，根据当前任务（task at hand）对其他模块进行配置——调制（modulate）它们的参数和注意力回路（attention circuits）。

它可以"引导"（prime）感知、世界模型和代价模块去聚焦特定目标。 这是执行控制（executive control）的核心：接到任务后，先配置其他模块，系统才开始感知-行动循环。

类比

前额叶皮层（prefrontal cortex）——负责任务切换、注意力分配、目标设定。这是当前 VLA 系统中基本缺失的模块。

所有模块的"总指挥" 调制参数 + 注意力

感知模块

PERCEPTION MODULE

接收传感器原始信号，估计世界的当前状态（current state of the world）。 对于给定任务，只有感知到的一小部分是相关的——Configurator 会引导感知系统只提取当前任务所需的信息。

感知模块可以以分层（hierarchical）方式表示世界状态，具备多个抽象层次。

对应 VLA

视觉编码器（SigLIP / ViT）+ 语言理解部分，输出状态表示 s[0] = Enc(x)。

可训练 输出：状态表示 s[0]

世界模型

WORLD MODEL MODULE

架构中最复杂的模块，功能双重：

① 补全缺失信息：估计感知模块未能提供的世界状态信息。
② 预测未来状态：给定 Actor 提议的动作序列，预测可能的未来世界状态 s[1], s[2]…

它用潜变量（latent variables）表示预测的不确定性，可输出多种可能的未来—— 这是 LeCun 强调的关键难题：自然世界的不可完全预测性。

关键难题

如何让 World Model 表示多种可能的预测？如何在高维连续空间中训练 World Model？这是 JEPA（Joint Embedding Predictive Architecture）试图回答的问题。

可训练 核心模块 当前 VLA 中隐式存在

代价模块

COST MODULE

用一个标量"能量"（energy）衡量智能体的"不适程度"（discomfort）。 由两个子模块组成：

内禀代价（Intrinsic Cost）：不可训练 / 硬编码
计算即时不适感，如痛苦（高能量）、愉悦（低/负能量）、饥饿等。 这是基本行为驱动（drives）和内在动机（intrinsic motivations）的来源。

可训练评论家（Trainable Critic）：可训练
预测未来的内禀代价，类似 RL 中的 value function / critic network。

类比

Intrinsic Cost ≈ 生物本能（痛觉、饥饿）；Critic ≈ 学习得来的价值判断。两者都可以反向传播梯度。

目标：最小化长期平均能量 可反向传播梯度

短期记忆

SHORT-TERM MEMORY MODULE

存储过去、当前和预测的世界状态，以及对应的内禀代价值。 World Model 在做时序预测或空间补全时会读写此模块； Critic 从中检索历史状态-代价对来训练自身。

架构类似 Key-Value Memory Networks， 可以视为海马体（hippocampus）在脊椎动物中的类比物。

读 / 写 类比：海马体

行动者模块

ACTOR MODULE

计算并输出动作序列提议。Actor 有两种工作模式：

① Policy 模式（System 1，直觉）：直接从 s[0] 计算 a[0] = A(s)，快速反应，不调用 World Model。

② 优化模式（System 2，慢思考）：提议动作序列 → World Model 预测未来状态 → Cost 评估能量 → 梯度优化动作序列（model-predictive control），找到最优动作，执行第一个。

对应 VLA

Action Expert / Flow Matching 输出头。π₀ 的 flow matching 可以视为 Policy 模式（System 1）的一种实现，但没有显式的 World Model 推演。

可训练 输出：action → 环境
§ 05

Mode-1：感知-动作片段

论文 Figure 3 展示了最简单的工作模式——反应式感知-动作片段（reactive perception-action episode）， 不调用 World Model，直接从感知输出动作：

x 原始观测 → Enc(x) 感知编码 → s[0] 世界状态 → A(s) Policy → a[0] 动作 → 环境

可选地，系统会用 Pred(s[0], a[0]) 预测下一状态 s[1]， 并用代价模块计算 C(s[0])，存入短期记忆。这样在下一次观测到来时， World Model 可以对齐校正自身。

System 1 vs System 2

Mode-1 对应 Kahneman 的"系统一"（System 1）——快速、直觉式、无需深度规划。 当 Actor 使用梯度优化找最优动作序列时，则对应"系统二"（System 2）。

§ 06

代价模块与驱动设计

LeCun 在论文中给出了一些 Intrinsic Cost 的设计例子，说明如何通过硬编码驱动来塑造行为：

低能量驱动（奖励）
保持直立 → 驱使机器人行走
当腿式机器人站立时能量低；跌倒时能量高。无需显式编程"学走路"。
低能量驱动（奖励）
感知他人喜悦 → 驱使共情行为
当附近人表现出快乐时能量低，驱使 agent 主动促进他人快乐。
低能量驱动（奖励）
遭遇新情境 → 驱使好奇心探索
新颖情境触发低能量，驱动探索，对应内在动机（intrinsic motivation）研究。
高能量（惩罚）
靠近危险 → 回避行为
靠近极端高温、火源等立即触发高能量，无需强化学习反复试错。

这种设计的优雅之处在于：不需要为每个任务单独设计奖励函数—— 只需设计底层驱动，具体的行为策略通过 Actor + World Model 的规划自然涌现。

§ 07

JEPA 深入：能量模型与非生成式预测

这是论文最具技术含量、也最常被单独引用的部分。LeCun 在这里给出了 World Model 的具体形式： JEPA（Joint-Embedding Predictive Architecture，联合嵌入预测架构）。 它是后续 I-JEPA（图像，2023）、V-JEPA（视频，2024）一系列工作的奠基蓝图。

一句话定义

JEPA = 不在像素空间预测未来，而是把输入 x 和未来 y 各自编码成抽象表示 s_x、s_y， 然后训练一个预测器 Pred 让 Pred(s_x, z) ≈ s_y，其中 z 是潜变量，吸收所有不可预测的细节。

7.1 能量模型（EBM）：统一框架

LeCun 把整个学习问题套进 EBM（Energy-Based Model）框架。EBM 不直接输出概率，而是给每对 (x, y) 一个标量"能量" F(x, y)： 兼容的对能量低，不兼容的对能量高。预测就是找让能量最小的 y。

概率模型必须归一化（partition function 难算），强迫给所有 y 配概率
EBM只需要相对能量：低能量 = "对的"，高能量 = "错的"，无需归一化
类比不要求"打分总和=100"，只要求"对的方案分数低、错的方案分数高"

7.2 三种架构对比

在能量模型框架下，"如何根据 x 预测 y" 有三种可能的架构。LeCun 力推第三种。

① GENERATIVE · 生成式
Decoder 从 x 直接生成 y
在像素/原始空间预测 ŷ = Dec(x, z)，能量 = ‖y − ŷ‖。
问题：逼 decoder 去精确画出树叶的每一根脉络、人脸的每一根头发——浪费容量在不可预测细节上。
② CONTRASTIVE · 对比式
把正样本拉近、负样本推开
SimCLR、MoCo 等。
问题：需要构造大量负样本（negative samples），高维空间下负样本数随维度指数增长，难 scale。
③ JEPA · 联合嵌入预测 ⭐
在抽象空间预测，无需重建
s_x = Enc(x), s_y = Enc(y), 训练 Pred(s_x, z) ≈ s_y。
优势：encoder 自动学会丢弃不可预测的低层细节；不需要负样本，靠正则项防坍缩。

7.4 防坍缩：VICReg 系正则

JEPA 的最大风险是表示坍缩（representational collapse）—— encoder 偷懒，输出常数向量，使 Pred 永远=s_y，loss=0 但什么也没学到。LeCun 提出三个方向的正则：

V · Variance每个嵌入维度方差 ≥ 阈值，强迫表示"撑开"
I · Invariance预测 s_y 与真实 s_y 距离最小化（主任务）
C · Covariance不同维度去相关，避免冗余

对应论文 §8 的 VICReg 训练目标，是 I-JEPA / V-JEPA 实现时的关键工程细节。另一类做法是 非对称结构（asymmetric）：给 Enc_y 加 stop-gradient 或用 EMA 目标网络（类似 BYOL）。

7.5 Hierarchical JEPA（H-JEPA）

单层 JEPA 只能做一步预测。要支持长时程规划（long-horizon planning）， LeCun 提议堆叠多层 JEPA：底层做短时间高细节的预测，高层在更抽象的表示上做长时间预测。

类比

做饭的规划：底层 JEPA 预测"下一秒手会到哪"，中层预测"切完菜后开火"，高层预测"30 分钟后端上桌"。 每往上一层，时间步长更大、细节更少、表示更抽象——这正是人类规划的层级结构。

7.6 为什么 LeCun 反对生成式 World Model

核心论点

像 Sora、DreamerV3 这类像素级生成世界模型，必须把容量花在头发、纹理、噪点这些 本质上不可预测的细节上。JEPA 的赌注是：智能体根本不需要在脑中"画出"未来， 只需要预测"未来在抽象层面会怎样"——这才是规划真正需要的信息。

LeCun 的批评

§ 09

批判与局限

争议

这篇报告没有实验验证，是纯粹的架构提案。批评者指出：
① 如何训练高维连续空间的 World Model 仍是开放问题；
② Intrinsic Cost 的设计本质上仍是人工，只是换了一层抽象；
③ 与当前基于 transformer + 大数据 scaling 的路线没有实质冲突，更像是不同抽象层级的描述。

尽管如此，这篇报告作为思考框架（conceptual framework）极有价值—— 它提供了一套语言来描述当前 AI 系统缺少什么，以及下一步研究的方向。
